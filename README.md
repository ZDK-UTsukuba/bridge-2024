# BRIDGE 2024年版

他学類の講義の学生による評価を確認できるシステム

## 使い方

BRIDGEサイト内「BRIDGEとは」をご覧ください。

## 開発者向け情報

### 開発を行うためには

```sh
git clone https://github.com/ZDK-UTsukuba/bridge-2024.git
cd bridge-2024
npm install
npm run dev
```

まずお手元のPC等でシェルを開き、上記を実行してください。正常にWebブラウザ等で開くことが出来るかどうかを確認してください。

### フレームワーク

フロントエンドのフレームワークは[Astro](https://astro.build)を採用しています。

#### CSSについて

フレームワークとして[Bulma v1](https://bulma.io)を使用している他、適宜SCSSを用いています。

### 主要なページ

主要なページとして

- `index`ページ
- BRIDGEについてのページ
- 科目ごとの詳細ページ
- ランキングページ

が存在します。

### 「似た評価の科目」の評価

各科目に関して以下の手法を用いて「似た評価の科目」の評価を行っています。各科目間の類似度は以下の要素を組み合わせて計算し、上位3件を表示しています。

#### 基本スコア

- **評価類似度** (40%): 単位の取りやすさ、オススメ度、授業の難易度などの評価項目の類似度をユークリッド距離により計算
- **時程類似度** (60%): 曜日・時限の一致度を計算（完全一致: 1.0、モジュール一致: 0.4、両方とも時程なし: 0.5）

#### 調整係数

- **タイプ一致ボーナス**: 同じタイプ（一般科目同士/体育科目同士）の場合、スコアを2.0倍
- **所属一致ボーナス**: 同じ所属（学類・学群等）の場合、スコアを1.5倍
- **評価数調整**: 評価数が多いほど高く評価するが、その速度を対数的に抑制。`(1 + log10(評価数 + 1)) / 1.3` により調整

#### 最終スコア

これらを掛け合わせたものが「最終スコア」となり、各科目ページにおいて上位3つが表示されています。

```
最終スコア = 基本スコア * タイプボーナス * 所属ボーナス * 評価数調整
```

より詳細な実装は`src/content/similarSubjects.ts`をご覧ください。
